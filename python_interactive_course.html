دورة بايثون

الدرس 1: المقدمة، الإعداد، والأساسيات

1. الشرح التفصيلي

مرحبًا بك في عالم بايثون! هذا القسم هو نقطة انطلاقك. سنتعلم كيف "نتحدث" مع الكمبيوتر باستخدام بايثون. سنبدأ بأبسط أمر وهو طباعة رسالة على الشاشة، ثم سنتعلم كيف نخزن المعلومات (البيانات) في الذاكرة باستخدام ما يسمى "بالمتغيرات".

المفاهيم الأساسية

  • دالة `print()`: هذه هي أداتك الأساسية لإظهار النتائج. هي دالة جاهزة تأخذ "وسيط" (Argument) - مثل نص أو رقم - وتطبعه على الشاشة. النص الذي نكتبه مباشرة يسمى "Literal".
  • المتغيرات (Variables): تخيلها كصناديق في ذاكرة الكمبيوتر. كل صندوق له اسم فريد (مثل `my_age`) ويحتوي على قيمة (مثل `30`). في بايثون، نحن لا نصرح عن نوع المتغير؛ بايثون تعرف النوع تلقائيًا (Dynamic Typing).
  • التعليقات (Comments): نستخدم علامة الـ `#` لكتابة ملاحظات للمبرمجين. المترجم (Interpreter) الخاص ببايثون يتجاهل أي شيء يأتي بعد `#` في نفس السطر.
  • أنواع البيانات الأساسية (Data Types):
    • النصوص (Strings): `str`. سلسلة من الأحرف. يجب وضعها بين علامتي تنصيص: `"Hello"` أو `'Hello'`.
    • الأرقام الصحيحة (Integers): `int`. أرقام كاملة بدون كسور (مثل: `5`, `-20`).
    • الأرقام العشرية (Floats): `float`. أرقام تحتوي على فاصلة عشرية (مثل: `3.14`, `-0.5`).
    • القيم المنطقية (Booleans): `bool`. تمثل حالتين فقط: `True` (صحيح) أو `False` (خطأ). (لاحظ الحرف الأول الكبير).
    • اللاشيء (NoneType): `None`. نوع بيانات خاص يمثل "الغياب" أو "اللا قيمة".

3. تريكات وحيل (Tricks & Tips)

تريكة 1: F-Strings (الأفضل لتنسيق النصوص)

بدلاً من استخدام `+` و `str()` لدمج النصوص والمتغيرات، استخدم `f-strings`. ضع حرف `f` قبل النص، وضع المتغيرات داخل `{}`.


name = "أحمد"

# لاحظ الـ f قبل علامة التنصيص

# يمكنك حتى إجراء عمليات داخلها
print(f"عمري بعد 5 سنوات سيكون {age + 5}")

المسألة 7: حاسبة فاتورة المطعم

اكتب برنامجًا كاملاً: 1. يسأل المستخدم عن سعر الوجبة (float). 2. يسأل المستخدم عن نسبة الإكرامية التي يود دفعها (مثلاً: 15). 3. يحسب قيمة الإكرامية (السعر * النسبة / 100). 4. يحسب الفاتورة الإجمالية (السعر + الإكرامية). 5. يطبع قيمة الإكرامية والإجمالي (منسقًا لرقمين عشريين).


# 1. الحصول على المدخلات
bill_str = input("أدخل سعر الوجبة: ")
tip_percentage_str = input("أدخل نسبة الإكرامية (مثلاً 15): ")

# 2. تحويل الأنواع
bill = float(bill_str)
tip_percentage = int(tip_percentage_str) # النسبة عادة رقم صحيح

# 3. الحسابات
tip_amount = bill * (tip_percentage / 100)
total_bill = bill + tip_amount

# 4. الطباعة (منسقة)
print("\n--- تفاصيل الفاتورة ---")
print(f"قيمة الإكرامية: {tip_amount:.2f}")
print(f"الإجمالي للدفع: {total_bill:.2f}")

الدرس 2: هياكل البيانات (Lists, Tuples, Sets, Dictionaries)

1. الشرح التفصيلي

هياكل البيانات (أو "المجموعات" - Collections) تسمح لنا بتخزين مجموعات من القيم في متغير واحد. كل هيكل له خصائصه وطريقة استخدامه.

  • القوائم (Lists): `[]`
    • هي حاويات مرتبة (Ordered) وقابلة للتعديل (Mutable).
    • نصل للعناصر عن طريق "الفهرس" (Index) الذي يبدأ من 0.
    • يمكننا تعديلها (إضافة، حذف، تغيير عناصر) في أي وقت. هي الأكثر شيوعًا.
  • Tuples: `()`
    • مثل القوائم، هي مرتبة، ولكنها غير قابلة للتعديل (Immutable).
    • بمجرد إنشائها، لا يمكن تغيير عناصرها. أسرع من القوائم وتستخدم للبيانات الثابتة.
  • القواميس (Dictionaries): `{}`
    • حاويات تخزن البيانات على هيئة مفتاح: قيمة (Key: Value).
    • قابلة للتعديل. نصل إلى القيم باستخدام "المفتاح" (Key) بدلاً من الفهرس.
  • المجموعات (Sets): `{}`
    • حاويات غير مرتبة وتضمن عدم تكرار العناصر.
    • فائدتها الرئيسية هي إزالة التكرارات وإجراء العمليات الرياضية (اتحاد، تقاطع).

2. أمثلة توضيحية

التعامل مع القوائم (Lists):


# إنشاء قائمة
my_list = ["apple", "banana", "cherry"]
print(f"القائمة: {my_list}")

# الوصول للعنصر الأول (الفهرس 0)
print(f"العنصر الأول: {my_list[0]}")

# الوصول للعنصر الأخير (الفهرس -1)
print(f"العنصر الأخير: {my_list[-1]}")

# إضافة عنصر (في النهاية)
my_list.append("orange")
print(f"بعد الإضافة: {my_list}")

# حذف عنصر (بمعرفة القيمة)
my_list.remove("banana")
print(f"بعد الحذف: {my_list}")

# حذف عنصر (بمعرفة الفهرس)
popped_item = my_list.pop(0) # حذف "apple" وإرجاعه
print(f"العنصر المحذوف: {popped_item}")
print(f"القائمة الآن: {my_list}")

3. تريكات وحيل (Tricks & Tips)

تريكة 1: التحقق من الوجود `in`

استخدم الكلمة المفتاحية `in` للتحقق مما إذا كان عنصر موجودًا في قائمة (بحث بال قيمة) أو مفتاح موجودًا في قاموس (بحث بالمفتاح).


my_list = [10, 20, 30]
print(f"هل 20 موجودة؟ {20 in my_list}") # True

student = {"name": "سارة", "age": 22}
print(f"هل 'age' مفتاح موجود؟ {'age' in student}") # True
print(f"هل 'سارة' موجودة؟ {'سارة' in student}") # False (لأن 'in' تبحث في المفاتيح)

تريكة 2: الوصول الآمن للقاموس `.get()`

إذا حاولت الوصول لمفتاح غير موجود `student['job']` سيتوقف البرنامج (Error). استخدم `.get()`، فإذا لم يجد المفتاح، سيعيد `None` (أو قيمة افتراضية) بدون إيقاف البرنامج.


student = {"name": "سارة", "age": 22}
# print(student['job']) # هذا سيسبب خطأ

# الطريقة الآمنة
job = student.get('job')
print(f"الوظيفة: {job}") # None

# الطريقة الآمنة مع قيمة افتراضية
job_default = student.get('job', 'طالب')
print(f"الوظيفة: {job_default}") # طالب

تريكة 3: إزالة التكرار بـ `set`

أسرع طريقة لإزالة التكرار من قائمة هي تحويلها إلى `set` ثم إعادتها إلى `list`.


numbers = [1, 2, 2, 3, 1, 4, 5, 2, 3]
unique_numbers = list(set(numbers))
print(unique_numbers) # [1, 2, 3, 4, 5] (الترتيب قد يختلف)

الدرس 3: العمليات والمنطق والشروط

1. الشرح التفصيلي

هذا هو جوهر البرمجة: جعل البرنامج يتخذ قرارات. نستخدم الشروط (Conditions) للتحكم في مسار تنفيذ الكود بناءً على ما إذا كان الشرط `True` (صحيح) أو `False` (خطأ).

المفاهيم الأساسية

  • عمليات المقارنة (Comparison Operators):
    • `==` (يساوي): هل 5 == 5؟ (True)
    • `!=` (لا يساوي): هل 5 != 10؟ (True)
    • `>` (أكبر من): هل 10 > 5؟ (True)
    • `<` (أصغر من): هل 10 < 5؟ (False)
    • `>=` (أكبر من أو يساوي)
    • `<=` (أصغر من أو يساوي)
  • أمر `if`: ينفذ الكود الذي يليه (المزاح بمسافة بادئة) فقط إذا كان الشرط `True`.
  • أمر `else`: (اختياري) ينفذ الكود الذي يليه إذا كان شرط `if` الرئيسي `False`.
  • أمر `elif`: (اختياري، اختصار لـ "else if") يسمح لك بإضافة شروط متعددة للتحقق منها بالترتيب.
  • العمليات المنطقية (Logical Operators):
    • `and`: يجب أن يكون كلا الشرطين `True` ليكون الناتج `True`.
    • `or`: يكفي أن يكون شرط واحد `True` ليكون الناتج `True`.
    • `not`: يعكس الحالة (يحول `True` إلى `False` والعكس).

3. تريكات وحيل (Tricks & Tips)

تريكة 1: القيم الصادقة والكاذبة (Truthy & Falsy)

في بايثون، لا تحتاج دائمًا لكتابة `if len(my_list) > 0`. يمكنك كتابة `if my_list:` فقط. بايثون تعتبر القوائم/النصوص الفارغة، الرقم `0`، و `None` كقيم `False` (Falsy). أي شيء آخر يعتبر `True` (Truthy).


my_name = input("أدخل اسمك (أو اتركه فارغًا): ")

# إذا كان النص my_name فارغًا ("")، سيعتبر Falsy
if my_name:
    print(f"مرحبًا، {my_name}")
else:
    print("مرحبًا أيها الغريب.")

تريكة 2: المقارنات المتسلسلة

بدلاً من كتابة `if x > 10 and x < 20`، يمكنك في بايثون كتابتها بشكل رياضي: `if 10 < x < 20`.


temp = 25
if 20 < temp < 30:
    print("الطقس دافئ ومثالي.")

تريكة 3: المعامل الثلاثي (Ternary Operator)

طريقة لاختصار `if/else` بسيط في سطر واحد.


age = 22
#       [القيمة إذا كان True] if [الشرط] else [القيمة إذا كان False]
status = "بالغ" if age >= 18 else "قاصر"
print(status)

الدرس 4: الحلقات التكرارية (Loops)

1. الشرح التفصيلي

الحلقات التكرارية تتيح لنا تنفيذ نفس الكتلة من الكود عدة مرات، مما يوفر علينا إعادة كتابة الكود. هناك نوعان رئيسيان في بايثون: `for` و `while`.

المفاهيم الأساسية

  • حلقة `for` (For Loops):
    • تُستخدم للتكرار على "تسلسل" (Sequence) معروف مسبقًا، مثل قائمة، tuple، نص، أو `range`.
    • تأخذ عنصرًا واحدًا من التسلسل في كل مرة وتنفذ الكود عليه.
    • دالة `range()`: تُستخدم لإنشاء تسلسل من الأرقام. `range(5)` تعني (0, 1, 2, 3, 4).
  • حلقة `while` (While Loops):
    • تستمر في التكرار طالما أن الشرط المحدد هو `True`.
    • نحن مسؤولون عن تغيير المتغير داخل الحلقة حتى يصبح الشرط `False` في النهاية، وإلا سندخل في "حلقة لا نهائية" (Infinite Loop).
  • أوامر التحكم بالحلقات:
    • `break`: يكسر الحلقة ويخرج منها فورًا.
    • `continue`: يتجاهل باقي الكود في هذه التكرارة الحالية وينتقل إلى التكرارة التالية.

2. أمثلة توضيحية

مثال على حلقة `for` (طباعة عناصر قائمة):


fruits = ["تفاح", "موز", "كرز"]

# 'fruit' هو متغير مؤقت يأخذ قيمة كل عنصر بالترتيب

    print(f"أنا أحب {fruit}")

# استخدام range() للطباعة 3 مرات
for i in range(3):
    print(f"الرقم {i}") # سيطبع 0, 1, 2

مثال على حلقة `while` (العد التنازلي):


count = 5

while count > 0:
    print(count)
    count = count - 1 # أو (count -= 1) - مهم جدًا لتجنب حلقة لا نهائية
    
print("انطلاق!")

مثال على `break` و `continue`:


# مثال break: توقف عند العثور على أول رقم سالب
numbers = [1, 5, 9, -2, 4, 8]
for num in numbers:
    if num < 0:
        print("تم العثور على رقم سالب، سنتوقف.")
        break
    print(f"الرقم {num} موجب.")

# مثال continue: طباعة الأرقام الفردية فقط
for i in range(1, 11): # الأرقام من 1 إلى 10
    if i % 2 == 0: # إذا كان الرقم زوجيًا
        continue # تخطى هذه التكرارة
    print(f"رقم فردي: {i}")

تريكة 3: `else` مع الحلقات

(نادرة الاستخدام) يمكن إضافة `else` في نهاية حلقة `for` أو `while`. يتم تنفيذها *فقط* إذا اكتملت الحلقة بشكل طبيعي (أي لم يتم إيقافها بـ `break`).

الدرس 5: الدوال (Functions)

1. الشرح التفصيلي

الدوال هي كتل بناء أساسية في البرمجة. إنها كتل من الكود منظمة وقابلة لإعادة الاستخدام، مصممة لأداء مهمة محددة. بدلاً من تكرار نفس الكود، نكتبه مرة واحدة داخل دالة ونقوم "باستدعائها" (Call) كلما احتجنا إليه.

المفاهيم الأساسية

  • التعريف (Defining): نستخدم الكلمة المفتاحية `def` متبوعة باسم الدالة وأقواس `()`.
  • الاستدعاء (Calling): نكتب اسم الدالة متبوعًا بالأقواس `()` لتنفيذ الكود بداخلها.
  • المعاملات (Parameters): هي المتغيرات التي نعرّفها داخل الأقواس `()` عند تعريف الدالة، لاستقبال قيم من الخارج.
  • الوسائط (Arguments): هي القيم الفعلية التي نمررها للدالة عند استدعائها.
  • القيمة المُرجعة (Return Value): نستخدم `return` لإخراج قيمة من الدالة. إذا لم نستخدم `return`، فإن الدالة تعيد `None` افتراضيًا.
  • القيم الافتراضية (Default Arguments): يمكننا إعطاء المعاملات قيمًا افتراضية تُستخدم إذا لم يمرر المستخدم قيمة عند الاستدعاء.
  • نطاق المتغيرات (Scope): المتغيرات المعرفة *داخل* الدالة (Local Scope) لا يمكن الوصول إليها من *خارج* الدالة.

2. أمثلة توضيحية

دالة بسيطة (بدون معاملات أو إرجاع):


# 1. تعريف الدالة
def greet():
    print("مرحبًا بك في بايثون!")
    print("الدوال تجعل الكود منظمًا.")

# 2. استدعاء الدالة (يمكن استدعاؤها عدة مرات)
greet()
greet()

دالة بمعاملات (Parameters) وإرجاع (Return):


# 'a' و 'b' هما معاملات (Parameters)
def add_numbers(a, b):
    total = a + b
    return total # إرجاع القيمة

# 5 و 10 هما وسائط (Arguments)
sum_result = add_numbers(5, 10)

print(f"الناتج هو: {sum_result}")
print(f"ناتج آخر: {add_numbers(100, 50)}")

دالة بقيمة افتراضية:


# "Guest" هي قيمة افتراضية للمعامل name
def welcome_user(name="Guest"):
    print(f"أهلاً بك، {name}")

welcome_user("أحمد") # استخدام القيمة الممررة
welcome_user()       # استخدام القيمة الافتراضية

3. تريكات وحيل (Tricks & Tips)

تريكة 1: سلاسل التوثيق (Docstrings)

هي أول شيء تكتبه داخل الدالة، محاطًا بثلاث علامات تنصيص `"""`. تُستخدم لشرح وظيفة الدالة، معاملاتها، وما تُرجعه. وهي ضرورية جدًا للاحترافية.


def say_hello(name):
    """
    تطبع رسالة ترحيب مخصصة.
    
    Args:
        name (str): اسم الشخص المراد تحيته.
    """
    print(f"مرحبًا، {name}!")

# يمكن للمحررات قراءة هذا التوضيح
say_hello("علي")

تريكة 2: تلميحات النوع (Type Hinting)

(اختياري) يمكن تحديد أنواع المعاملات والقيمة المرجعة المتوقعة. هذا لا يغير سلوك الكود، ولكنه يساعد المحررات والمبرمجين الآخرين على فهم الكود.


# (name: str) -> str
# يعني: المعامل 'name' يتوقع أن يكون 'str'
# و (-> str) تعني: الدالة يتوقع أن ترجع 'str'
def greet(name: str) -> str:
    return f"Hello, {name}"

print(greet("World"))

تريكة 3: إرجاع قيم متعددة

يمكن للدالة إرجاع قيم متعددة (والتي يتم إرجاعها فعليًا كـ `Tuple`).


def get_user_info():
    name = "Ali"
    age = 33
    return name, age # إرجاع قيمتين

# استقبال القيمتين
user_name, user_age = get_user_info()
print(f"{user_name} is {user_age} years old.")

4. مسائل تطبيقية (متدرجة الصعوبة)

المستوى 1: الأساسيات

المسألة 1: طباعة رسالة

المسألة 6: دالة تتعامل مع قائمة

أنشئ دالة `find_largest_number` تأخذ قائمة أرقام، وتُرجع أكبر رقم فيها. (استخدم دالة `max()` الجاهزة داخل دالتك).


def find_largest_number(numbers):
    """يرجع أكبر رقم من قائمة. يرجع None إذا كانت القائمة فارغة."""
    if not numbers: # تريكة (Falsy) للتحقق إذا كانت القائمة فارغة
        return None
    return max(numbers)

print(find_largest_number([10, 5, 80, 25])) # 80
print(find_largest_number([]))             # None

المسألة 8: دالة بإرجاع متعدد

أنشئ دالة `get_min_max` تأخذ قائمة أرقام. الدالة يجب أن تُرجع (في أمر `return` واحد) كلاً من *أصغر* رقم (`min()`) و *أكبر* رقم (`max()`).


def get_min_max(numbers):
    """يرجع أصغر وأكبر رقم في قائمة."""
    if not numbers:
        return None, None # إرجاع Tuple من None
        
    smallest = min(numbers)
    largest = max(numbers)
    return smallest, largest # إرجاع قيمتين (كـ Tuple)

my_nums = [10, 5, 80, 25]
# استقبال القيمتين
min_val, max_val = get_min_max(my_nums)

print(f"الأصغر: {min_val}, الأكبر: {max_val}")

المسألة 9: دمج الدوال

استخدم الدالة `is_even` (من م4) و `add_numbers` (من م2). أنشئ دالة ثالثة `sum_if_even` تأخذ رقمين، إذا كان *كلاهما* زوجي (استخدم `is_even`)، تُرجع مجموعهما (استخدم `add_numbers`)، وإلا تُرجع 0.


# نفترض أن الدالتين معرفتان مسبقًا
def add_numbers(a, b):
    return a + b

def is_even(number):
    return number % 2 == 0

# --- الدالة الجديدة ---
def sum_if_even(num1, num2):
    if is_even(num1) and is_even(num2):
        return add_numbers(num1, num2)
    else:
        return 0

print(f"ناتج (4, 6): {sum_if_even(4, 6)}") # 10
print(f"ناتج (4, 5): {sum_if_even(4, 5)}") # 0

المسألة 10: مشروع (حاسبة المتوسط)

أنشئ دالة `calculate_average` تأخذ قائمة درجات. 1. احسب المجموع (استخدم `sum()`). 2. احسب العدد (استخدم `len()`). 3. تُرجع المتوسط (المجموع / العدد). 4. (تحدي إضافي): تأكد من أن القائمة ليست فارغة قبل القسمة.


def calculate_average(grades):
    """يحسب متوسط قائمة من الدرجات."""
    # (التحدي الإضافي)
    if not grades:
        return 0.0 # لا يمكن القسمة على صفر
        
    total = sum(grades)
    count = len(grades)
    average = total / count
    return average

grades_list = [80, 90, 100, 85]
avg = calculate_average(grades_list)
print(f"متوسط الدرجات هو: {avg}")
